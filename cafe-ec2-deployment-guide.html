☁️ Phase 1 — AWS EC2 Setup

Do all of this in the AWS Management Console from your computer's browser.

1 Launch an EC2 Instance

  1. Log in to AWS Console → go to EC2 → click Launch Instance
  2. Name: type cafe-management-server
  3. AMI (OS): Select Amazon Linux 2 AMI (HVM)
    Look for the one that says "Amazon Linux 2" — it's usually the first option and marked "Free tier eligible"
  4. Instance Type: Select t2.medium
    ⚠️
    Do NOT use t2.micro (Free Tier) for this project. Building Angular + Spring Boot requires at least 2GB RAM. t2.micro (1GB) will run out of memory and crash during the build. t2.medium costs ~$0.05/hour — stop the instance when not using it.
  5. Key Pair: Click Create new key pair
    • Name: cafe-key
    • Type: RSA
    • Format: .pem (for Mac/Linux) or .ppk (for Windows with PuTTY)
    • Click Create key pair — it downloads automatically. Save it safely!
  6. Network settings: Click Edit
    • VPC: leave default
    • Auto-assign public IP: Enable
    • Firewall: Create new security group → name it cafe-sg
  7. Storage: Change from 8 GB to 20 GB (gp2)
    Docker images + build cache can easily use 10+ GB
  8. Click Launch Instance → wait ~1 minute → click the instance ID to open it
  9. Copy the Public IPv4 address shown (e.g., 54.123.45.67) — you'll use this everywhere
🟠
Write down your EC2 Public IP! You'll need it in multiple places: SSH connection, .env file, and to access the app in browser. It changes every time you stop and start the instance (unless you use an Elastic IP).

2 Configure Security Group (Open Ports)

Security Groups are EC2's firewall. You need to open exactly these ports:

  1. In EC2 Console → left sidebar → Security Groups
  2. Click on cafe-sg
  3. Click Inbound rules tab → Edit inbound rules
  4. Add these rules (click Add rule for each):
TypeProtocolPortSourcePurpose
SSHTCP22 My IP (auto-fills your IP) Connect to EC2 from your computer
HTTPTCP80 0.0.0.0/0 (Anywhere) Users access the web app
✅
That's all you need to open! Port 8081 (backend), 3306 (MySQL), and 6379 (Redis) stay closed to the internet. Nginx handles forwarding inside Docker.

Click Save rules.

3 Connect to EC2 via SSH

From your local computer's terminal, connect to the EC2 instance.

On Mac / Linux:

# First: fix permissions on your key file (required by SSH)
chmod 400 ~/Downloads/cafe-key.pem

# Connect to EC2 (replace with your actual Public IP)
ssh -i ~/Downloads/cafe-key.pem ec2-user@54.123.45.67

# Type "yes" when asked about fingerprint

On Windows (using PowerShell):

# PowerShell supports ssh natively
ssh -i C:\Users\YourName\Downloads\cafe-key.pem ec2-user@54.123.45.67

On Windows (using PuTTY):

  1. Open PuTTYgen → Load your .pem file → Save as .ppk
  2. Open PuTTY → Host: ec2-user@54.123.45.67 → Port: 22
  3. Connection → SSH → Auth → Browse → select your .ppk file → Open
💡
You should see a prompt like [ec2-user@ip-172-xx-xx-xx ~]$ — this means you are inside your EC2 server. All following commands are typed here.

🐳 Phase 2 — Install Docker on EC2

Run all commands below in your SSH terminal (connected to EC2).

4 Install Docker & Docker Compose

Amazon Linux 2 does not come with Docker. Run these commands one by one:

# Step 4a: Update the system
sudo yum update -y

# Step 4b: Install Docker
sudo yum install docker -y

# Step 4c: Start Docker and enable it to start on reboot
sudo systemctl start docker
sudo systemctl enable docker

# Step 4d: Allow your user to run Docker without "sudo" every time
sudo usermod -aG docker ec2-user

# Step 4e: IMPORTANT — log out and log back in so the group change takes effect
exit

# Now SSH back in again:
ssh -i ~/Downloads/cafe-key.pem ec2-user@54.123.45.67
# Step 4f: Install Docker Compose v2
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-linux-x86_64" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Step 4g: Also make "docker compose" (with space) work
sudo mkdir -p /usr/local/lib/docker/cli-plugins
sudo curl -SL "https://github.com/docker/compose/releases/latest/download/docker-compose-linux-x86_64" -o /usr/local/lib/docker/cli-plugins/docker-compose
sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

# Step 4h: Verify both work
docker --version
docker compose version

# Expected output:
# Docker version 24.x.x
# Docker Compose version 2.x.x
✅
If docker ps works without needing sudo, Docker is correctly installed.

5 Upload Your Project Files to EC2

You need to transfer your project files from your local computer to EC2. You have two options:

Option A — Upload using SCP (recommended)

Run this from your local computer's terminal (not inside SSH):

# Upload the frontend zip

unzip ~/Cafe-Management-System-FE-main.zip -d /tmp/

📝 Phase 3 — Code Changes & Configuration Files

All the following commands are run inside your EC2 SSH session in the ~/cafe-management folder.

6 Fix the Source Code

Three files must be fixed before Docker can build successfully.

Fix 1 — CafeConstants.java (Windows path → Linux path)

nano ~/cafe-management/backend/src/main/java/com/in/cafe/constants/CafeConstants.java
📄 Replace the entire file content with this:
package com.in.cafe.constants;

public class CafeConstants {

    public static final String SOMETHING_WENT_WRONG = "Something Went Wrong.";
    public static final String INVALID_DATA         = "Invalid Data.";
    public static final String UNAUTHORIZED_ACCESS   = "Unauthorized access.";

    // ✅ Linux path — works in Docker on EC2
    public static final String STORE_LOCATION = "/app/cafe-stored-files";
}

Save: Ctrl+X → Y → Enter

Fix 2 — BillServiceImpl.java (backslash → forward slash)

Use sed to find-and-replace the backslash path separators automatically:

sed -i 's|STORE_LOCATION + "\\\\" +|STORE_LOCATION + "/" +|g' \
  ~/cafe-management/backend/src/main/java/com/in/cafe/serviceImpl/BillServiceImpl.java

# Verify the fix worked (should show "/" not "\\"):
grep "STORE_LOCATION" ~/cafe-management/backend/src/main/java/com/in/cafe/serviceImpl/BillServiceImpl.java

Fix 3 — environment.ts (hardcoded IP → empty for Nginx proxy)

cat > ~/cafe-management/frontend/src/environments/environment.ts << 'EOF'
export const environment = {
  production: false,
  apiUrl: ''
};
EOF

cat > ~/cafe-management/frontend/src/environments/environment.prod.ts << 'EOF'
export const environment = {
  production: true,
  apiUrl: ''
};
EOF
💡
Why empty string? With apiUrl: '', all Angular HTTP calls go to the same host the browser is on. Nginx then sees those requests and forwards the API ones to the backend container. The browser never needs to know the backend's IP.

Fix 4 — application.properties (Redis host fix)

The properties file has spring.data.redis.host=localhost — inside Docker, Redis is reachable by its container name, not localhost. Fix it:

sed -i 's/spring.data.redis.host=localhost/spring.data.redis.host=redis/' \
  ~/cafe-management/backend/src/main/resources/application.properties

# Verify:
grep "redis.host" ~/cafe-management/backend/src/main/resources/application.properties
# Should show: spring.data.redis.host=redis

7 Create Backend Dockerfile

cat > ~/cafe-management/backend/Dockerfile << 'EOF'
# Stage 1 — Build .jar with Maven
FROM maven:3.9.6-eclipse-temurin-17 AS build
WORKDIR /app
COPY . .
RUN mvn clean package -DskipTests

# Stage 2 — Run .jar in lightweight Java container
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
RUN mkdir -p /app/cafe-stored-files
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8081
ENTRYPOINT ["java", "-jar", "app.jar"]
EOF

8 Create Frontend Dockerfile

cat > ~/cafe-management/frontend/Dockerfile << 'EOF'
# Stage 1 — Build Angular app
FROM node:16-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install --legacy-peer-deps
COPY . .
RUN npm run build -- --configuration=production

# Stage 2 — Serve with Nginx
FROM nginx:alpine
COPY --from=build /app/dist/frontend /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
EOF

9 Create Nginx Configuration

This config serves Angular and proxies all API calls to the backend container:

cat > ~/cafe-management/frontend/nginx.conf << 'EOF'
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    # Angular routing — refresh on any URL works
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy all API calls to Spring Boot backend
    location ~ ^/(user|category|product|bill|dashboard|password)/ {
        proxy_pass       http://cafe-app-backend:8081;
        proxy_set_header Host            $host;
        proxy_set_header X-Real-IP       $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 10M;
    }
}
EOF

10 Create docker-compose.yml

cat > ~/cafe-management/docker-compose.yml << 'EOF'
version: '3.8'

services:

  mysql:
    image: mysql:5.7
    container_name: mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASS}
      MYSQL_DATABASE:      cafe_db
      MYSQL_USER:          ${DB_USER}
      MYSQL_PASSWORD:      ${DB_PASS}
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - cafe_network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-uroot", "-p${DB_ROOT_PASS}"]
      interval: 10s
      timeout: 5s
      retries: 10

  redis:
    image: redis:7-alpine
    container_name: redis
    restart: always
    networks:
      - cafe_network

  cafe-app-backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: cafe-app-backend
    restart: always
    environment:
      DB_URL:            jdbc:mysql://mysql:3306/cafe_db?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true
      DB_USER:           ${DB_USER}
      DB_PASS:           ${DB_PASS}
      MAIL_USER:         ${MAIL_USER}
      APP_PASS:          ${APP_PASS}
      ALLOWED_ORIGIN:    ${ALLOWED_ORIGIN}
      FRONTEND_URL:      ${FRONTEND_URL}
    volumes:
      - cafe_bills:/app/cafe-stored-files
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_started
    networks:
      - cafe_network

  cafe-app-frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: cafe-app-frontend
    restart: always
    ports:
      - "80:80"
    depends_on:
      - cafe-app-backend
    networks:
      - cafe_network

networks:
  cafe_network:
    driver: bridge

volumes:
  mysql_data:
  cafe_bills:
EOF
🔐
Port exposure change from local setup: On EC2, the backend (8081), MySQL (3306), and Redis (6379) are NOT exposed to the host. Only port 80 is. This is intentional — those services are only accessible inside the Docker network, not from the internet.

🔧 Troubleshooting EC2-Specific Issues

❌ Page won't load — "This site can't be reached"

# 1. Check Security Group has port 80 open (Step 2)
# 2. Check frontend container is running:
docker compose ps
docker compose logs cafe-app-frontend

❌ "docker: command not found" after installing

# You need to log out and back in after the usermod command
exit
# Then SSH back in and try again

❌ Build fails — "Killed" or "Out of memory"

# This means your EC2 instance doesn't have enough RAM
# Solution: Add swap space (virtual memory)
sudo dd if=/dev/zero of=/swapfile bs=128M count=16
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile swap swap defaults 0 0' | sudo tee -a /etc/fstab

# Verify swap is active:
free -h

# Now retry the build:
cd ~/cafe-management && docker compose up --build -d

❌ Backend keeps restarting

# Check what error it's throwing:
docker compose logs cafe-app-backend

# Common cause: MySQL not ready yet. Wait 30 seconds then:
docker compose restart cafe-app-backend

# Another common cause: wrong DB credentials in .env
cat ~/cafe-management/.env

❌ Angular API calls fail — CORS error in browser console

# Check ALLOWED_ORIGIN in .env matches your EC2 IP exactly (no trailing slash):
cat ~/cafe-management/.env | grep ALLOWED_ORIGIN
# Must be: ALLOWED_ORIGIN=http://54.123.45.67  (your actual IP)

# Rebuild after fixing .env:
docker compose down
docker compose up --build -d

❌ EC2 Public IP changed after restart

# Your IP changes every time you stop/start the instance.
# Option 1: Update .env with the new IP and redeploy
# Option 2 (permanent fix): Allocate an Elastic IP in AWS Console
#   EC2 → Elastic IPs → Allocate → Associate with your instance
#   This IP never changes (free while instance is running)

❌ "port 80 already in use" on EC2

# Check what's using port 80:
sudo ss -tlnp | grep :80

# If Apache/httpd is running, stop it:
sudo systemctl stop httpd
sudo systemctl disable httpd

# Then retry:
docker compose up -d

❌ PDF bills not generating

# Verify the code fix in Step 6 was applied:
grep "STORE_LOCATION" ~/cafe-management/backend/src/main/java/com/in/cafe/serviceImpl/BillServiceImpl.java
# Must show "/" not "\\"

# Check the volume directory exists inside the container:
docker exec cafe-app-backend ls /app/cafe-stored-files

❌ Storage running low on EC2

# Check disk usage:
df -h

# Clean up Docker build cache:
docker system prune -f

# See what's using space in Docker:
docker system df
